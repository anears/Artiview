# Artiview

**코딩 에이전트가 쏟아내는 HTML 결과물을 위한 로컬 갤러리.**

[English guide](README.md)

![Artiview 데모 — 라이브러리 탐색, 전문 검색, 문서 내 검색, 라이트/다크 테마](docs/demo.gif)

AI 에이전트(Claude Code, Codex, 사내 파이프라인…)는 HTML 리포트·발표자료·
Markdown 문서를 디스크 곳곳에 — 그리고 서버에도 — 쌓아 놓습니다. Artiview는
그 폴더들을 탐색 가능한 라이브러리로 바꿔줍니다: 라이브 렌더 썸네일, 본문
전문 검색, 즐겨찾기·태그, 그리고 로컬 폴더와 똑같이 동작하는 SSH/SFTP 원격
폴더까지.

앱을 관통하는 세 가지 약속:

- **원본 파일을 절대 수정·삭제하지 않습니다.** Artiview는 읽고 색인만 합니다.
  앱 안의 모든 "제거"는 색인 항목만 지웁니다.
- **모든 데이터는 로컬에 있습니다.** 색인은 내 컴퓨터의 SQLite 파일 하나이고,
  SSH 비밀번호는 디스크에 기록되지 않습니다.
- **신뢰할 수 없는 문서는 샌드박스에서 렌더링됩니다.** 에이전트가 만든 HTML은
  `allow-same-origin` 없는 iframe에서 열리므로 악성 문서가 앱이나 시스템에
  접근할 수 없습니다.

## 설치

[Releases](https://github.com/anears/Artiview/releases)에서 플랫폼에 맞는
최신 빌드를 내려받으세요:

| 플랫폼 | 파일 | 참고 |
|---|---|---|
| macOS (Apple Silicon) | `Artiview_x.y.z_aarch64.dmg` | 미서명 앱: 첫 실행 시 **우클릭 → 열기**, 또는 시스템 설정 → 개인정보 보호 및 보안에서 허용. |
| Windows | `Artiview_x.y.z_x64-setup.exe` 또는 `.msi` | SmartScreen 경고가 뜨면 "추가 정보 → 실행"을 선택하세요. |
| Linux | `.AppImage`, `.deb`, `.rpm` | AppImage는 `chmod +x` 후 실행. |

Windows/Linux 빌드는 최근 추가되어 테스트가 충분하지 않습니다 — 문제가 있으면
[이슈로 알려주세요](https://github.com/anears/Artiview/issues).

UI는 기본 영어이며 시스템 언어가 한국어면 자동으로 한국어가 됩니다. 라이트/
다크 테마는 시스템을 따릅니다. 둘 다 언제든 바꿀 수 있습니다 —
[설정](#설정) 참고.

## 처음 5분

1. **폴더를 등록하세요.** 사이드바 *폴더* 옆 **+** 를 누르고 에이전트 결과물이
   쌓이는 디렉터리(예: `~/agent-reports`)를 고릅니다. 하위 폴더까지 포함해
   모든 `.html`, `.htm`, `.md` 파일이 스캔·색인됩니다.
2. **그리드가 채워지는 걸 보세요.** 각 카드는 실제 문서를 썸네일 크기로 라이브
   렌더링한 것입니다 — 오래된 스크린샷이 아닙니다.
3. **그냥 켜 두세요.** 등록한 로컬 폴더는 자동으로 감시됩니다. 에이전트가 새
   파일을 만들거나 기존 파일을 다시 쓰면 몇 초 안에 라이브러리에 알아서
   나타납니다. 새로고침 버튼이 필요 없습니다.
4. **카드를 클릭**하면 내장 뷰어로 문서를 읽을 수 있습니다. `Esc`로
   돌아옵니다.

등록 폴더 밖의 파일을 하나만 보고 싶다면? 우측 상단 **파일 열기**로 바로 열 수
있고, *최근 본 파일*에 기록됩니다.

## 라이브러리 탐색

- **보기** — 사이드바 상단에서 *전체 / 최근 본 파일 / 즐겨찾기* 를 전환합니다.
- **폴더 트리** — 등록한 루트가 하위 폴더로 펼쳐집니다. 클릭하면 그 아래
  파일만 보입니다.
- **태그** — 사이드바의 태그를 클릭하면 해당 태그로 필터링됩니다.
- **그리드 / 목록** — ▦ / ☰ 버튼으로 전환. 목록 보기는 크기와 정확한 시각을
  보여줍니다.
- **정렬** — 툴바에서 *수정일 · 이름 · 크기 · 생성일 · 열어본 날짜* 를 고르고
  ↑/↓로 방향을 바꿉니다. 선택은 재시작 후에도 유지됩니다.
- 파일 이름은 자동으로 정해집니다: `<title>` 태그 → 첫 헤딩 → 파일명.

## 무엇이든 찾기

검색은 두 겹입니다:

- **라이브러리 검색** (툴바) — 파일명뿐 아니라 **모든 문서의 본문 텍스트**를
  검색합니다. 파일명이 `output_final_v3.html`이어도 본문에 "매출"이 있으면
  `매출`로 찾을 수 있습니다.
- **문서 내 검색** — 뷰어에서 `⌘/Ctrl+F`. 일치 항목이 그 자리에서 하이라이트
  되고, `Enter` / `Shift+Enter`로 오가며 `Esc`로 닫습니다.

## 문서 읽기

파일을 클릭하면 뷰어가 열립니다:

| 컨트롤 | 동작 |
|---|---|
| ⌕ 찾기 | 문서 내 검색 (`⌘/Ctrl+F`) |
| # 태그 | 태그 추가/제거 |
| ★ | 즐겨찾기 토글 |
| 🗑 제거 | 라이브러리에서 잊기 (디스크의 파일은 그대로) |
| 브라우저로 ↗ | 기본 브라우저로 원본 열기 (로컬 파일) |
| `Esc` | 라이브러리로 돌아가기 |

Markdown은 GitHub 스타일 + 코드 하이라이트로 렌더되고, 문서 안의 상대경로
이미지·링크는 파일이 있는 폴더 기준으로 표시됩니다 — 원격 서버의 파일도
마찬가지입니다.

## 즐겨찾기와 태그로 정리하기

- 카드·목록·뷰어 어디서든 **★** 를 누르면 *즐겨찾기*에 고정됩니다.
- 뷰어의 **# 태그**로 태그를 붙이면 사이드바에 개수와 함께 나타나 원클릭
  필터가 됩니다.
- 즐겨찾기와 태그는 재스캔, 파일 이동, 드라이브 분리에도 살아남습니다.

## 서버의 폴더 쓰기 (SSH/SFTP)

에이전트가 원격 머신에서 돌고 있다면, 그 결과물 폴더를 SSH로 등록해 로컬처럼
쓰세요:

1. *폴더* 옆 **🌐** 를 클릭합니다.
2. **접속 대상** — `user@host`, `user@host:포트`, 또는 `~/.ssh/config`의 별칭.
3. **원격 경로** — 절대 경로를 입력하기 시작하면 하위 폴더가 제안됩니다.
   `Tab`은 셸처럼 완성, `↑`/`↓`로 선택, `Enter`로 확정.
4. **인증** — *자동*은 ssh-agent → ssh-config의 `IdentityFile` → 기본 키
   (`id_ed25519` 등) 순으로 시도합니다. 키 파일(`.pem`)이나 비밀번호를 직접
   지정할 수도 있습니다.
5. **추가** — 첫 스캔이 바로 시작됩니다.

검색·썸네일·뷰어·태그가 로컬 폴더와 완전히 동일하게 동작하고, 원격 HTML 안의
이미지·CSS는 SFTP로 스트리밍됩니다. 원격 폴더는 사이드바에서 🌐 아이콘과
`@호스트` 표시로 구분됩니다.

**비밀번호에 대해:** 앱 실행 중에만 메모리에 유지됩니다. 재시작 후 잠긴
호스트는 🔒 카드로 표시되며, 클릭해서 비밀번호를 입력하면 다시 연결됩니다.
원격 폴더는 **↻ 새로고침** 버튼으로 갱신합니다 (변경분만 재스캔).

## 파일이 없어졌을 때

파일은 이동·이름변경·삭제되고, 드라이브는 뽑히고, 서버는 내려갑니다.
Artiview는 성급하게 지우지 않습니다:

- 없어진 파일은 **배지로 표시**될 뿐 자동 삭제되지 않습니다 — 태그·즐겨찾기가
  유지됩니다.
- 뷰어에서 **다시 시도**(일시적 문제)나 **라이브러리에서 제거**(의도적 정리)를
  고를 수 있습니다.
- 드라이브나 서버가 통째로 사라져도 항목은 보존되고, 복구된 뒤 첫 스캔에서
  원래대로 돌아옵니다.

## 설정

사이드바 하단의 **⚙ 설정**:

- **언어** — 자동(시스템) / English / 한국어
- **테마** — 자동(시스템) / 라이트 / 다크

확인을 누르면 적용됩니다. 보기 방식과 정렬은 자동으로 기억되므로 따로 설정할
것이 없습니다.

## 키보드 단축키

| 키 | 동작 |
|---|---|
| `⌘/Ctrl + F` | 열린 문서에서 검색 |
| `Enter` / `Shift+Enter` | 다음 / 이전 일치 항목 |
| `Esc` | 검색 바 → 대화상자 → 뷰어 순으로 닫기 |
| `Tab` | 원격 경로 자동완성 (원격 폴더 추가 대화상자) |

## 자주 묻는 질문 · 문제 해결

**원본 파일이 수정되거나 삭제되나요?**
아니요, 절대로요. 읽기만 합니다. 모든 제거 동작은 색인에만 적용됩니다.

**데이터는 어디에 저장되나요?**
폴더·색인·최근·즐겨찾기·태그는 플랫폼별 앱 데이터 디렉터리의
`library.db`(SQLite)에 저장됩니다. SSH 비밀번호는 어디에도 저장되지 않습니다.

**원격 폴더가 "인증 실패"라고 나와요.**
먼저 터미널에서 `ssh 대상`으로 접속되는지 확인하세요. ssh-agent에 키가 있거나
(`ssh-add -l`) `~/.ssh/config`에 `IdentityFile`이 지정돼 있으면 *자동*이
찾아냅니다. 키 파일 권한은 `600`이어야 합니다.

**스캔이 일부 파일을 건너뛰어요.**
원격 스캔은 안전을 위해 깊이 16단계 / 항목 2만 개 / 파일당 50MB로 제한됩니다.

**문서가 비어 보이거나 깨져요.**
**브라우저로 ↗** 로 브라우저 렌더링과 비교하고, 뷰어의 **다시 시도**를
눌러보세요. 외부 네트워크 리소스에 의존하는 문서는 오프라인에서 다르게 보일
수 있습니다.

## 알려진 제한

- Windows/Linux 빌드는 최근 추가되어 테스트가 충분하지 않습니다 — 이슈 제보
  환영합니다. macOS는 Apple Silicon 빌드만 제공됩니다.
- SSH host key를 아직 `known_hosts`와 대조하지 않습니다 — 신뢰할 수 있는
  네트워크에서 사용하세요.
- SSH 비밀번호는 앱 재시작 시 다시 입력해야 합니다 (Keychain 연동 예정).
- 파일 접근 범위는 `tauri.conf.json`의 `assetProtocol.scope`로 제한됩니다
  (기본: 홈 디렉터리와 이동식 미디어 마운트).

## 개발자용

```bash
npm install
npm run tauri dev     # 개발 실행
npm run tauri build   # 현재 플랫폼용 릴리스 번들 빌드
```

```
src-tauri/           Rust 백엔드 (Tauri 2)
  src/db.rs          SQLite 스키마 + FTS5 + 쿼리
  src/html.rs        <title>/헤딩/본문 텍스트 추출
  src/remote.rs      SSH/SFTP 연결 풀 + remote:// 프로토콜 서버
  src/watch.rs       네이티브 파일 감시 → 디바운스 증분 재스캔
  src/lib.rs         폴더 스캔 + Tauri 커맨드
src/                 React 프론트엔드
  api.ts             invoke 래퍼 + 로컬/원격 URL 변환
  i18n.ts            UI 문자열 (기본 영어, 시스템 언어가 한국어면 한국어)
  markdown.ts        markdown-it + highlight.js 렌더 + iframe 소스 훅
  components/        Sidebar · Toolbar · FileGrid · FileCard · Viewer · 모달들
```

뷰어·썸네일 iframe은 `allow-same-origin` 없이 샌드박스되어 신뢰할 수 없는
문서가 앱 권한에 접근할 수 없습니다. 원격 콘텐츠는 커스텀 `remote://`
프로토콜이 SFTP를 프록시하며 asset protocol과 동일한 CORS 정책을 따릅니다.
CI는 모든 PR에서 macOS/Linux/Windows 번들을 빌드하고, `v*` 태그를 푸시하면
번들이 첨부된 드래프트 릴리스를 만듭니다.
